# Troubleshooting Framework (Fast + Documentable)

1. **Confirm Impact**
   - Who is affected? One user or many?
   - What is the deadline/urgency (class starting, meeting in progress)?

2. **Gather Symptoms**
   - Exact error messages, when it started, what changed.

3. **Check the Obvious (Quick Wins)**
   - Power/cables, correct input/output device, mute buttons, airplane mode, etc.

4. **Isolate the Domain**
   - Device vs Network vs Account vs App vs AV hardware.

5. **Test a Theory**
   - Run targeted checks (commands/logs/settings).

6. **Apply Fix (Least-Risk First)**
   - Reversible changes first.

7. **Verify**
   - Re-test original workflow; confirm with user.

8. **Document**
   - Steps taken, root cause, resolution, time, prevention, escalation decision.
